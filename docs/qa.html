Frequently Asked Questions ​

  • Integrate the NetGuard SDK into the client app, not the server app.
  • Due to OS limitations, NetGuard does not support forwarding to ports below 1024 (i.e., 0-1023).
  • Due to OS limitations, mobile local loopback addresses must be 127.0.0.1.
  • To avoid conflicts when multiple PC apps use NetGuard, different local loopback addresses can be used.
  • For multiple processes of the same PC app, integrate NetGuard in a separate process.
  • If the PC app crashes after initializing the SDK, use separate processes for integration.
  • For compliance reasons, NetGuard does not support:
    1. Forwarding data to servers within LAN.
    2. Forwarding data to servers within China.
    3. Assigning Forwarding Servers to clients running within China.
  • To connect clients in China to the NetGuard server, please Customize the Domain Name
